Text Generation Application using NLP and LLM

Link to GitHub

The project is a text generation application that processes accident descriptions using Natural Language Processing (NLP) and Large Language Model (LLM) techniques.

The text generation application is structured around two main components:

  • Text Processing
  • Text Generation

The process begins with text processing, where raw text is cleaned and prepared. Then, text generation uses the processed text as a base to produce generated content.

Below is the general architecture of the application development in Visual Studio Code. I divided my code into two main parts:

  • Frontend: Developed using Flask
  • Backend: Developed in Python with the following components:
    • Error Correction Stage
    • Information Extraction Stage
    • General Processing Stage
    • Translation Stage
    • Filtering Stage
    • Generation Stage

Accident Report Processing Pipeline

1. Information Extraction Stage

This first stage involves extracting the date and location of the accident from the description. It is considered the initial step to ensure that these essential pieces of information are not removed during the subsequent text cleaning process.

  • Date extraction is performed using the regex library.
  • Location extraction is achieved through Named Entity Recognition (NER) in NLP.

2. Text Cleaning Stage

Text cleaning is an essential step in natural language processing. The cleaning process involves removing:

  • URLs
  • Email addresses
  • Arbitrary numbers
  • HTML tags
  • Any other unnecessary information

This stage also includes removing words that are not found in the dictionary, using the BERT model from Hugging Face.

3. Text Correction Stage

For the text correction stage, a function is used to correct typographical errors, as well as grammar and spelling mistakes. This is done using specific tools depending on the language:

  • Pyspellchecker for French
  • TextBlob for English

4. Text Translation Stage

The translation process starts by translating individual words from a file named car-accident-translation, which contains a list of French words and their English translations related to accidents. If a word is not found in this file, the Google Translate API is used. If the API translation is insufficient for the context, the pre-trained MarianMTModel from Hugging Face is used for more accurate translation.

5. Text Filtering Stage

The objective of this stage is to remove all irrelevant or out-of-context sentences using a classification model. The model identifies and filters out non-pertinent sentences in the text. The steps are as follows:

  • Data Collection and Labeling
  • Data Splitting, Preprocessing, and Tokenization using DistilBertTokenizerFast
  • Model Configuration and Fine-Tuning with DistilBERT

6. Extraction of Vehicle Type and Sentiments

The extraction of the vehicle type and sentiment is performed after the text correction and translation to ensure accurate identification of vehicles in English. I provided a file listing all vehicle types in English for reliable extraction.

Sentiment extraction is done using a Hugging Face pipeline for sentiment analysis.

7. Text Generation Stage

This stage uses a pretrained generation model to produce texts based on a given prompt.

  • distilbert/distilgpt2 from Hugging Face Transformers
