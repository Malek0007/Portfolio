Plant Disease Detection

Link to GitHub

Presentation of the project

The project is a plant disease detection application that can identify diseases or anomalies based on images of leaves.

Technologies Used

  • Deep Learning: Using TensorFlow and Keras
  • HTML,CSS and Javascript: For the frontend
  • Flask: For the backend
  • MLOps: For deploying the application

Functionalities

This model is prepared towards building a complete machine learning pipeline for image classification, from dataset preparation to model training, evaluation, and prediction.

Data preparation and preprocessing:

The first step is preparation and preprocessing of data, which includes key tasks like resizing the images to a uniform dimension, rescaling pixel values to a range between 0 and 1, and applying data augmentation to artificially expand the dataset. Data augmentation introduces random transformations such as flipping, rotating, and zooming. Additionally, the model optimizes the TensorFlow pipeline by using techniques like caching and prefetching.Thus improving overall training performance.

Training the deep learning model:

The model is a Convolutional Neural Network (CNN) built with TensorFlow's Keras API, starting with resizing, rescaling, and data augmentation. It uses six Conv2D layers, each followed by MaxPooling2D to reduce spatial dimensions while capturing important features. The model applies ReLU activation and ends with a Dense layer of 64 units, followed by a softmax output layer for multi-class classification (15 classes). It is compiled with the Adam optimizer and Sparse Categorical Crossentropy, and trained using model.fit(), with accuracy used for evaluation.

Developing the application and deploying it:

In this stage, we can make predictions and then save the model as a .keras file for later use. This model will be integrated into the Flask application to provide prediction capabilities. After integrating the model, we can containerize the application using Docker to ensure consistent deployment across different environments. This approach simplifies the management of dependencies and makes scaling the application easier
